### Приветствую, человек 🤖

### GitHub Copilot CLI - 1.0.52
- Добавлена вертикальная полоса прокрутки с поддержкой перетаскивания мышью в основном представлении диалога.
- Выбор уровня контекстного окна (по умолчанию ~200K против 1M токенов) теперь применяется комплексно, ограничивая сжатие, обрезку и отображение токенов.
- Пользовательские агенты поддерживают опциональную отложенную загрузку инструментов через `deferred-tool-loading` во фронтматерии агента, что позволяет обнаруживать инструменты для агентов с большими списками инструментов.
- `/compact` принимает опциональные инструкции фокуса для формирования сводки сжатия.
- Субагенты общего назначения используют GPT-5.4 или GPT-5.5, когда доступны.
- `/usage` показывает индикаторы прогресса квоты для лимитов сеанса и недели.
- Сообщения об ошибках AI credits обновлены более четким языком и ссылкой на управление бюджетом.
- Переключение в режим Autopilot больше не вызывает неожиданные запросы разрешений для доступа к инструментам, путям или URL.
- `copilot --continue` из сохраненной директории сеанса теперь обновляет сохраненную ветку и git-контекст вместо того, чтобы оставлять их устаревшими.
- Сеансы теперь возобновляются в сохраненной рабочей директории; передайте `-C <dir>` для переопределения.
- Reasoning tokens отображаются в скобках в счетчике выходных токенов в сводке использования токенов.
- Команда строки состояния поддерживает обычные shell-команды в дополнение к путям исполняемых скриптов.
- Автоматическая очистка старых файлов логов процессов из `~/.copilot/logs/` при запуске для предотвращения неограниченного роста диска.

### GitHub Copilot CLI - 1.0.53
- Многострочные промпты отображаются полностью без обрезки контента или смещения выделения.
- Выбор `/skills` теперь корректно учитывает `--config-dir` при сохранении предпочтений навыков.
- Сеансы Bash shell больше не зависают, когда в окружении установлены PS0 или PROMPT_COMMAND.

### GitHub Copilot CLI - 1.0.54
- Исправления и изменения.

### GitHub Copilot CLI - 1.0.55
- Увеличена контрастность фона выделения во всех цветовых темах для лучшей видимости.
- `/env` теперь показывает загруженные расширения с их статусом и источником.
- Терминальный звонок больше не звучит при завершении хода, если не включен явно через конфигурацию.
- Выбор `/resume` больше не показывает пустые строки для сеансов, закрытых до отправки сообщения.
- Прерывание сеанса больше не оставляет UI застрявшим в состоянии отмены, когда запущены агенты Task tool.
- Вызовы инструмента vote_memory ограничены по частоте на ответ и взаимодействие для предотвращения лавинных всплесков голосования.
- Автопрокрутка вверх теперь активируется при перетаскивании выделения мышью за верхнюю часть временной шкалы.
- Буфер обмена корректно копирует символы CJK и дополнительной плоскости Unicode на Windows.
- Расширения запускаются корректно, когда CLI работает как однофайловое исполняемое приложение (SEA).

### Visual Studio Code - 1.122
- Добавлено действие "Добавить скриншот в чат" во встроенном браузере для захвата и прикрепления скриншота текущей страницы к сообщению в чате.
- Всплывающие подсказки в списке сеансов окна Agents показывают полное название сеанса и путь к папке.
- Сохранение поля `gallery` при установке MCP-сервера через протокольный URL `vscode:mcp/install`, обеспечивая сохранение метаданных серверов из реестров для проверки обновлений и активации.
- MCP OAuth поддерживает указание пользовательского `clientId` в секции `oauth` файла `mcp.json`, что позволяет работать с серверами, не поддерживающими Dynamic Client Registration.
- Модели Bring Your Own Key (BYOK) теперь работают в изолированных сценариях без аутентификации GitHub.
- Локальный хост агента теперь включен по умолчанию в сборках Insiders.
- Выбор reasoning effort теперь показывает допустимый уровень вместо 'undefined' для семейств моделей, не указывающих явное значение по умолчанию.
- Редактор Language Models показывает детальные действия для групп провайдеров, такие как обновление API-ключа, добавление модели, переход к файлу конфигурации, переименование и удаление.
- Терминальные команды, выполняемые в чате, показывают точечные/ASCII анимации загрузки.
- Используйте `/models` во вводе чата для открытия выбора модели.
- Агенты теперь могут запускать задачи на удаленных машинах.
- Состояние системы контроля версий в окне Agents теперь обновляется автоматически после того, как агент делает коммит, синхронизацию или выполняет другие git-операции.
